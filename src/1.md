Уровень 1: Основы

1Создай ArrayList<Integer>, добавь 10 чисел через цикл, выведи все элементы.
2Дан ArrayList<String> с именами. Проверь, содержится ли в нём конкретное имя (contains), и выведи его индекс (indexOf).
3Удали из ArrayList элемент по конкретному значению и отдельно — по индексу. Покажи разницу между remove(Object) и remove(int).
4Дан ArrayList<Integer>. Посчитай сумму всех элементов и их среднее арифметическое.

Уровень 2: Циклы и условия

5Найди max и min элементы в ArrayList<Integer> без использования Collections.max/min — только циклом.
6Удали все четные числа из ArrayList<Integer> (сначала попробуй неправильно — через for-each — увидишь ошибку, потом исправь).
7Реализуй разворот списка "вручную" — обменивая элементы по индексам, без Collections.reverse().
8Дан ArrayList<Integer>. Раздели его на два списка: чётные и нечётные числа.

Уровень 3: Строки и работа с текстом

9Дан ArrayList<String> со словами. Найди самое длинное и самое короткое слово.
10Дан ArrayList<String>. Преобразуй все строки в верхний регистр и отфильтруй те, что короче 3 символов.
11Удали дубликаты из ArrayList<String>, сохранив порядок появления (сначала вручную, потом через LinkedHashSet — сравни подходы).
12Посчитай частоту встречаемости каждого слова в ArrayList<String> с помощью HashMap<String, Integer>.

Уровень 4: Алгоритмы

13Есть два отсортированных ArrayList<Integer>. Слей их в один отсортированный список (merge из merge sort), без вызова sort() в конце.
14Реализуй бинарный поиск элемента в отсортированном ArrayList<Integer> (без Collections.binarySearch).
15Найди вторую по величине уникальную сумму... нет, проще: найди второй максимальный элемент в ArrayList<Integer> (с учётом дубликатов и без них — два варианта).
16Дан ArrayList<Integer>. Найди все пары элементов, сумма которых равна заданному числу N (классическая задача "two sum" на списках).

Уровень 5: "Подкапотные" вопросы

17Покажи на практике ConcurrentModificationException — удали элемент из ArrayList во время for-each. Потом исправь двумя способами: через Iterator.remove() и через removeIf().
18Реализуй свой класс MyArrayList<T> с методами add, get, remove, size на основе обычного массива, с автоматическим увеличением (resize) при заполнении. Это один из самых частых вопросов на junior — проверяет понимание реализации ArrayList.
19Сравни производительность ArrayList и LinkedList: вставь 100 000 элементов в начало каждого списка, замерь время через System.nanoTime(), объясни разницу через сложность операций (O(1) vs O(n)).
20Дан ArrayList<Object> (или List<List<Integer>>) — список списков (как двумерный массив, но "рваный"). Выведи все элементы в одну строку, найди сумму всех чисел во всех вложенных списках (часто встречается как введение в работу с вложенными коллекциями перед Stream API).
